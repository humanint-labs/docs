Capisum provides official client libraries to reduce boilerplate, surface type errors at compile time, and get you from zero to your first API call as fast as possible. The Node.js / TypeScript SDK is the primary library and ships with full TypeScript definitions for every request shape, response object, and error class. If you're working in a language without a dedicated SDK, the REST API is always available — you can call it directly from any HTTP client without installing anything.

## Node.js / TypeScript SDK

### Installation

Install `@capisum/sdk` from npm using your preferred package manager:

<CodeGroup>
```bash npm
npm install @capisum/sdk
```
```bash yarn
yarn add @capisum/sdk
```
```bash pnpm
pnpm add @capisum/sdk
```
</CodeGroup>

### Initialization

Import the default export and instantiate the client with your API key. Pull the key from an environment variable rather than hard-coding it — this keeps it out of source control and lets you switch between test and production keys without touching your code.

```typescript
import Capisum from '@capisum/sdk';

const capisum = new Capisum({
  apiKey: process.env.CAPISUM_API_KEY,
});
```

<Tip>
  Use `cap_sk_test_...` keys during development to run full integration tests against the sandbox without moving real funds. Swap in your live key only when you're ready to go to production.
</Tip>

### Creating a Transaction

Call `capisum.payments.create()` with the provider, transaction type, amount, currency, and customer details. The SDK returns a fully typed response object — no casting required.

```typescript
const transaction = await capisum.payments.create({
  provider: "paystack",
  type: "pay_in",
  amount: 500000,
  currency: "NGN",
  customer: {
    email: "customer@example.com",
    name: "Jane Doe",
  },
  reference: "order-001",
});

console.log(transaction.id);     // txn_8f2k9m
console.log(transaction.status); // successful
```

`amount` is expressed in the smallest unit of the currency (kobo for NGN, cents for USD, and so on). The `provider` field tells Capisum which adapter to route the request through — you're in control of routing on every call.

### Retrieving a Transaction

Fetch a single transaction by its Capisum ID at any point after creation to check its current status or access the normalized response fields.

```typescript
const transaction = await capisum.transactions.get("txn_8f2k9m");
```

---

### Listing Transactions

Query your transaction history with filters. Results are paginated; use `limit` to control page size.

```typescript
const transactions = await capisum.transactions.list({
  status: "settled",
  provider: "paystack",
  limit: 20,
});
```

Supported filter fields include `status`, `provider`, `type` (pay_in / payout), and date range parameters. See the [API reference](/api-reference) for the full list.

### Handling Errors

Wrap calls in a `try/catch` block and check for `Capisum.APIError` to access the structured error envelope. Every error carries a stable `code` (e.g. `INSUFFICIENT_FUNDS`, `PROVIDER_TIMEOUT`) and a `retryable` boolean so your retry and alerting logic can stay provider-agnostic.

```typescript
try {
  const transaction = await capisum.payments.create({ ... });
} catch (error) {
  if (error instanceof Capisum.APIError) {
    console.log(error.code);      // INSUFFICIENT_FUNDS
    console.log(error.retryable); // false
  }
}
```

<Note>
  `retryable: true` indicates a transient failure — network timeouts, provider unavailability — where a retry is safe. `retryable: false` indicates a terminal failure like invalid credentials or insufficient funds where retrying without fixing the underlying issue will not help.
</Note>

## Using the REST API Directly

You don't need an SDK to use Capisum. Every operation available in the SDK is also available as a plain HTTP request. This means you can integrate from any language, script, or tool that can send an HTTP request — including shell scripts, Postman collections, and server-side frameworks in languages that don't yet have a dedicated client library.

<Note>
  Full endpoint documentation, request schemas, and example responses are available in the [API reference](/api-reference). Every example in the reference includes a raw HTTP snippet alongside the SDK equivalent.
</Note>

The base URL for all API requests is `https://api.capisum.com/v1`. Authenticate by passing your API key in the `Authorization` header as a Bearer token:

```bash
curl --request POST \
  --url https://api.capisum.com/v1/payments \
  --header "Authorization: Bearer $CAPISUM_API_KEY" \
  --header "Content-Type: application/json" \
  --data '{
    "provider": "paystack",
    "type": "pay_in",
    "amount": 500000,
    "currency": "NGN",
    "customer": {
      "email": "customer@example.com",
      "name": "Jane Doe"
    },
    "reference": "order-001"
  }'
```
