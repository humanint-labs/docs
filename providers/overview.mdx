Capisum gives you a single API surface to connect your application to 30+ fintech providers across payment gateways, digital banks and wallets, crypto platforms, and account infrastructure. Instead of maintaining separate integrations for each provider — each with its own request shapes, auth flows, and error formats — you write once against the Capisum API and switch or combine providers through configuration.

<Note>
  **Capisum is an integration layer, not a payment processor.** You need your own account and API credentials with each provider you intend to use. Capisum does not hold funds, process transactions on your behalf, or replace your provider agreements. It standardizes how your application talks to those providers.
</Note>

## Supported operations by category

Each provider category maps to a set of operations that Capisum normalizes across all providers in that group. You use the same request shape regardless of which provider you select.

| Category | Supported operations |
|---|---|
| Payment Gateways | Pay-in, Payout, Accounts (virtual accounts) |
| Digital Banks & Wallets | Wallet top-up, Pay-in, Payout |
| Crypto Platforms | Crypto buy/sell, Settlement, Wallet |
| Account Infrastructure | Account data, Account provisioning, Pay-by-bank |

## Provider status labels

Every provider in the Capisum network carries one of three status labels. You'll see these on each provider's detail page and in the dashboard.

| Status | Meaning |
|---|---|
| **Live** | Production-ready. The adapter is fully implemented and available for live traffic. |
| **Beta** | Available now with limited testing. You can connect and build against this provider, but the adapter may still be receiving improvements. |
| **Coming Soon** | On the roadmap. The provider is planned but not yet available for connection. |

## How to connect a provider

Before you can route transactions through a provider, you need to link your credentials inside the Capisum dashboard. The steps below apply to every provider regardless of category.

<Steps>
  <Step title="Add credentials in the dashboard">
    Go to **Settings → Providers** in the Capisum dashboard and select the provider you want to connect. Enter the API keys or credentials from your provider account. Capisum stores these securely and uses them when routing requests on your behalf.
  </Step>
  <Step title="Test in sandbox mode">
    Switch your Capisum environment to sandbox and run a test transaction. The sandbox forwards requests to the provider's own test environment (where available), so you can verify the full request-response cycle before going live.
  </Step>
  <Step title="Activate for production">
    Once your sandbox tests pass, flip the environment to production in the dashboard. Your live provider credentials will be used for all subsequent requests. You can connect multiple providers and choose the active one per transaction using the `provider` field in your API calls.
  </Step>
</Steps>

<Tip>
  You can connect multiple providers at the same time and select the provider per transaction using the `provider` field in your request payload. This gives you full control over routing without any automatic switching — smart routing is a planned future feature.
</Tip>
