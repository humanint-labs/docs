Capisum is a unified API for connecting and operating multiple financial providers through one consistent integration.

Instead of building and maintaining separate integrations for every payment gateway, bank, wallet, or financial infrastructure provider, you connect your existing provider accounts to Capisum using your own credentials.

Your application sends one standardized request. Capisum translates it into the selected provider's format, normalizes the response, tracks the transaction, and delivers consistent webhook events to your server.

Capisum does not hold your funds or replace your providers. It removes provider-specific complexity so your team can add, switch, and manage providers without rebuilding your payment infrastructure.

## Available capabilities

The API is available at `https://api.capisum.com/v1`. Payouts are the primary transaction type.

| Capability | Endpoint |
| --- | --- |
| Create a payout | `POST /v1/transactions/payout` |
| List and query transactions | `GET /v1/transactions` |
| Verify account names | `POST /v1/verify/account` |
| Check provider health and balance | `GET /v1/providers/{slug}/status`, `GET /v1/providers/{slug}/balance` |
| Receive status webhooks | Configure in the [dashboard](https://app.capisum.com/webhooks/config) |
