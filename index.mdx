Capisum gives your engineering team a single API to send transactions, process payouts, and manage accounts across payment gateways, digital banks, wallets, and crypto platforms. Instead of building and maintaining a separate integration for each provider, you connect once to Capisum and route requests to any supported provider with the same request shape, response format, and error codes.

## How Capisum Works

<Steps>
  <Step title="Connect a provider">
    Add your provider credentials (Paystack, Flutterwave, Kuda, Quidax, etc.) through the Capisum dashboard. Your credentials are stored securely and never exposed to your application code.
  </Step>
  <Step title="Send a request">
    Call `POST /v1/transactions` with the provider, amount, currency, and operation type. Capisum validates and normalizes the request before forwarding it.
  </Step>
  <Step title="Capisum routes to the provider">
    Capisum's adapter layer translates your request into the provider's native API format, handles authentication, and forwards the call.
  </Step>
  <Step title="Receive a standard response">
    Capisum returns a consistent response object regardless of which provider executed the transaction — same fields, same status vocabulary, every time.
  </Step>
</Steps>
