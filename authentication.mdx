The **public integrator API** (`/v1/*`) uses organization API keys. Create and revoke keys in the [dashboard](https://app.capisum.com).

<Info>
  Dashboard session APIs (`/api/v1/*`) use short-lived JWTs from token exchange. Those are for the Capisum web app — not for server-to-server integration.
</Info>

## API key format

```
cps_8Kx9mN2pQ7rT4vW1yZ3aB5cD6eF0gH
```

Keys use the `cps_` prefix. The full key is shown once at creation.

## Authorization header

```bash
curl "$CAPISUM_BASE_URL/v1/transactions" \
  -H "Authorization: Bearer cps_your_key_here"
```

| Header | Required | Description |
| --- | --- | --- |
| `Authorization` | Yes | `Bearer cps_...` |
| `Content-Type` | For JSON bodies | `application/json` |
| `X-Idempotency-Key` | For payout creation | See [Idempotency](/guides/idempotency) |

## Environment scoping

Each key is bound to **staging** or **production** at creation. The key determines which provider credentials and transaction records are used. You cannot override environment per request.

See [Environments](/guides/environments).

## Errors

| HTTP status | Code | Meaning |
| --- | --- | --- |
| 401 | `UNAUTHORIZED` | Missing or invalid API key |
