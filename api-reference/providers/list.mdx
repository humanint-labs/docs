The `/v1/providers` endpoint returns the full catalog of payment gateways, digital banks, wallets, crypto platforms, and account infrastructure providers that Capisum supports. You can narrow results by passing one or more query parameters — useful for building dynamic provider-selection UIs, validating routing decisions before sending transactions, or checking which providers are live versus still in beta.

## Endpoint

```
GET https://api.capisum.com/v1/providers
```

## Query Parameters

<ParamField query="category" type="string">
  Filter the list to a specific provider category. Accepted values:

  - `payment_gateway` — Traditional card and bank-transfer gateways (e.g., Paystack, Flutterwave, Monnify)
  - `digital_bank_wallet` — Digital banks and mobile wallet providers (e.g., Kuda, OPay, PalmPay)
  - `crypto_platform` — Crypto buy/sell and wallet platforms (e.g., Quidax, Binance, Luno)
  - `account_infrastructure` — Account provisioning and open-banking providers (e.g., Mono, Anchor, Bridge)
</ParamField>

<ParamField query="status" type="string">
  Filter by availability status. Accepted values:

  - `live` — Fully available and ready to use in production
  - `beta` — Available with limited functionality or under active development
  - `coming_soon` — On the roadmap but not yet accepting connections
</ParamField>

<ParamField query="operation" type="string">
  Filter to providers that support a specific operation type. Accepted values:

  - `pay_in` — Accepting inbound payments (cards, bank transfers, mobile money)
  - `payout` — Sending disbursements or transfers
  - `accounts` — Provisioning or managing virtual/bank accounts
  - `wallet` — Wallet top-ups, balance management, and merchant flows
  - `crypto` — Crypto buy, sell, and settlement workflows
</ParamField>

## Response Fields

<ResponseField name="data" type="array">
  Array of provider objects matching your query. Returns all providers when no filters are applied.

  <Expandable title="Provider object">
    <ResponseField name="id" type="string">
      The provider's unique slug. Use this value as the `provider` field when creating transactions (e.g., `paystack`, `flutterwave`, `kuda`).
    </ResponseField>

    <ResponseField name="name" type="string">
      Human-readable display name for the provider (e.g., `"Paystack"`, `"Flutterwave"`).
    </ResponseField>

    <ResponseField name="category" type="string">
      The category this provider belongs to. One of `payment_gateway`, `digital_bank_wallet`, `crypto_platform`, or `account_infrastructure`.
    </ResponseField>

    <ResponseField name="status" type="string">
      Current availability of the provider. One of `live`, `beta`, or `coming_soon`.
    </ResponseField>

    <ResponseField name="operations" type="array of strings">
      List of operation types this provider supports (e.g., `["pay_in", "payout"]`). Only operations in this list can be routed to the provider.
    </ResponseField>

    <ResponseField name="currencies" type="array of strings">
      ISO 4217 currency codes accepted by this provider (e.g., `["NGN", "USD", "GHS"]`). Sending a transaction in an unsupported currency will return a validation error.
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="total" type="integer">
  Total number of providers returned after applying any filters.
</ResponseField>

## Code Examples

<CodeGroup>

```bash cURL
# List all live payment gateways
curl "https://api.capisum.com/v1/providers?category=payment_gateway&status=live" \
  -H "Authorization: Bearer cap_sk_live_..."
```

```typescript TypeScript
const providers = await capisum.providers.list({
  category: "payment_gateway",
  status: "live",
});

for (const provider of providers.data) {
  console.log(provider.id, provider.operations);
}
```

</CodeGroup>
