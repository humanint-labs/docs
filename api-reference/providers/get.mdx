The `/v1/providers/{id}` endpoint returns the full detail record for a single provider, identified by its slug. In addition to the fields available in the list response, this endpoint surfaces two account-scoped fields: `connected` tells you whether your Capisum account has an active connection to the provider, and `sandbox_available` tells you whether you can run test transactions against it without going live. Use this endpoint to inspect provider capabilities before building a routing decision into your application.

## Endpoint

```text
GET https://api.capisum.com/v1/providers/{id}
```

## Path Parameters

<ParamField path="id" type="string" required>
  The provider slug. You can find valid slugs in the `id` field of the [List Providers](/api-reference/providers/list) response. Examples: `paystack`, `flutterwave`, `kuda`, `quidax`.
</ParamField>

## Response Fields

<ResponseField name="id" type="string">
  The provider's unique slug (e.g., `"paystack"`). This is the value you pass as the `provider` field in transaction requests.
</ResponseField>

<ResponseField name="name" type="string">
  Human-readable display name for the provider (e.g., `"Paystack"`).
</ResponseField>

<ResponseField name="category" type="string">
  The category this provider belongs to. One of `payment_gateway`, `digital_bank_wallet`, `crypto_platform`, or `account_infrastructure`.
</ResponseField>

<ResponseField name="status" type="string">
  Current availability of the provider across the Capisum network. One of `live`, `beta`, or `coming_soon`.
</ResponseField>

<ResponseField name="operations" type="array">
  List of operation types this provider supports (e.g., `["pay_in", "payout"]`). Attempting to send a transaction with an operation not in this list will result in a validation error.
</ResponseField>

<ResponseField name="currencies" type="array">
  ISO 4217 currency codes accepted by this provider (e.g., `["NGN"]`). Transactions in unsupported currencies will be rejected before reaching the provider.
</ResponseField>

<ResponseField name="connected" type="boolean">
  Whether your Capisum account currently has an active connection to this provider. Only connected providers can process transactions. Connect or manage providers from the Capisum dashboard.
</ResponseField>

<ResponseField name="sandbox_available" type="boolean">
  Whether sandbox mode is available for this provider. When `true`, you can send test transactions using a sandbox API key without triggering real payments on the provider side.
</ResponseField>

## Code Examples

<CodeGroup>

```bash cURL
curl https://api.capisum.com/v1/providers/paystack \
  -H "Authorization: Bearer cap_sk_live_..."
```

```typescript TypeScript
const provider = await capisum.providers.get("paystack");
console.log(provider.connected);         // true
console.log(provider.operations);        // ["pay_in", "payout"]
console.log(provider.sandbox_available); // true
```

</CodeGroup>

## Example Response

```json
{
  "id": "paystack",
  "name": "Paystack",
  "category": "payment_gateway",
  "status": "live",
  "operations": ["pay_in", "payout"],
  "currencies": ["NGN"],
  "connected": true,
  "sandbox_available": true
}
```

## Error Response — Provider Not Found

If you pass an unrecognized slug, Capisum returns a `404` with the following error shape:

```json
{
  "error": {
    "code": "NOT_FOUND",
    "message": "Provider 'unknown_provider' not found.",
    "retryable": false
  }
}
```

<Tip>
  Before routing a transaction to a provider, call this endpoint and verify that `connected` is `true`. If `connected` is `false`, the transaction will fail immediately — connect the provider from your dashboard first, then retry.
</Tip>
