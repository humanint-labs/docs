The Capisum public API lives under `/v1` and authenticates with organization API keys. All request and response bodies are JSON unless noted otherwise.

<Tip>
  New to Capisum? Start with the [Quickstart](/quickstart), then use this page as a map to endpoint docs in the sidebar.
</Tip>

## Authentication

```
Authorization: Bearer cps_your_api_key
```

See [Authentication](/authentication) for key format and [Environments](/guides/environments) for staging vs production scoping.

## Conventions

| Convention | Detail |
| --- | --- |
| Base URL | `https://api.capisum.com` (production) |
| Prefix | `/v1` for integrator endpoints |
| Success wrapper | `{ "data": ... }` for single resources |
| List pagination | `{ "data": [], "hasMore", "nextCursor", "prevCursor" }` — see [Pagination](/guides/pagination) |
| Timestamps | ISO 8601 UTC (`created_at`, `updated_at`) |
| Amounts | Integers in minor currency units — see [Payouts](/guides/payouts#amounts) |
| Idempotency | `X-Idempotency-Key` required for payout creation — see [Idempotency](/guides/idempotency) |

## Shipped endpoints

### Providers

| Method | Path | Description |
| --- | --- | --- |
| `GET` | `/v1/providers` | List available provider slugs |
| `GET` | `/v1/providers/{slug}/status` | Health snapshot for a provider |

<Note>
  `GET /v1/providers` returns the provider **catalog**, not your connected providers. See [FAQ](/guides/faq).
</Note>

### Transactions

| Method | Path | Description |
| --- | --- | --- |
| `POST` | `/v1/transactions/payout` | Create a payout |
| `GET` | `/v1/transactions` | List transactions (cursor pagination) |
| `GET` | `/v1/transactions/ref/{reference}` | Get by Capisum reference |
| `GET` | `/v1/transactions/{id}` | Get by transaction ID |
| `POST` | `/v1/transactions/{id}/verify` | Poll provider for latest status |

### Incoming webhooks

Called by payment providers, not your application:

| Method | Path | Description |
| --- | --- | --- |
| `POST` | `/v1/webhooks/fincra/{organizationSlug}/{environment}` | Fincra status webhooks |

See [Webhooks](/guides/webhooks) for setup instructions.

## Not in this API

The following are **not** exposed on `/v1` and are intentionally excluded from this reference:

- Dashboard session APIs (`/api/v1/auth/*`, `/api/v1/organizations/*`, etc.)
- API key management (`/api/v1/api-keys/*`)
- Provider credential storage and connection (`/api/v1/providers/*` dashboard routes)
- Routing rule CRUD (`/api/v1/routing/*`)
- Billing, audit logs, team management, and dashboard analytics
- Capisum internal admin APIs (`/api/v1/internal/*`)

Manage those capabilities in the [Capisum dashboard](https://app.capisum.com).

## OpenAPI

Endpoint pages in the sidebar are generated from `openapi.json`, synced from the Capisum API implementation via `capisum-api/scripts/generate-public-openapi.py`. Use the interactive playground on each endpoint page to send authenticated requests.
