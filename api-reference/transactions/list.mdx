Use this endpoint to retrieve a paginated list of transactions on your Capisum account. You can filter results by status, provider, operation type, your own reference string, or a datetime range — making it straightforward to build reconciliation reports, power dashboard views, or audit specific transaction cohorts. Results are returned in reverse chronological order and paginated using a cursor, so you can walk through large sets without skipping or duplicating records.

## Endpoint

```text
GET https://api.capisum.com/v1/transactions
```

## Query Parameters

<ParamField query="status" type="string">
  Filter transactions by their current lifecycle status. Accepted values: `pending`, `processing`, `successful`, `settled`, `failed`. Omit this parameter to return transactions across all statuses.
</ParamField>

<ParamField query="provider" type="string">
  Filter transactions by the provider slug that executed them (e.g., `paystack`, `flutterwave`, `kuda`). Must match a provider slug exactly.
</ParamField>

<ParamField query="type" type="string">
  Filter by operation type. Accepted values: `pay_in` or `payout`.
</ParamField>

<ParamField query="reference" type="string">
  Filter by your unique reference string. Because references are unique per transaction, this effectively looks up a single record — useful when you know your reference but not the Capisum transaction ID.
</ParamField>

<ParamField query="from" type="string">
  ISO 8601 datetime string defining the start of a date range filter on `created_at` (e.g., `2024-01-01T00:00:00Z`). Inclusive.
</ParamField>

<ParamField query="to" type="string">
  ISO 8601 datetime string defining the end of a date range filter on `created_at` (e.g., `2024-01-31T23:59:59Z`). Inclusive. Must be later than `from` when both are provided.
</ParamField>

<ParamField query="limit" type="integer" default={20}>
  Number of transaction records to return per page. Minimum `1`, maximum `100`. Defaults to `20`.
</ParamField>

<ParamField query="cursor" type="string">
  Pagination cursor returned as `next_cursor` in the previous response. Pass this value to retrieve the next page of results. Omit on the first request.
</ParamField>

## Code Examples

<CodeGroup>

```bash cURL
curl "https://api.capisum.com/v1/transactions?status=settled&provider=paystack&limit=20" \
  -H "Authorization: Bearer cap_sk_live_..."
```

```typescript TypeScript
const result = await capisum.payments.list({
  status: "settled",
  provider: "paystack",
  limit: 20,
});

for (const txn of result.data) {
  console.log(txn.id, txn.status);
}

// Get next page
if (result.has_more) {
  const nextPage = await capisum.payments.list({
    cursor: result.next_cursor,
    limit: 20,
  });
}
```

</CodeGroup>

## Response

A successful request returns a `200 OK` with a paginated envelope containing an array of transaction objects and cursor metadata for navigating additional pages.

<ResponseField name="data" type="array">
  Array of transaction objects matching the applied filters. Each object in the array represents a single transaction and contains the same fields returned by `GET /v1/transactions/:id`.

  <Expandable title="transaction object fields">
    <ResponseField name="id" type="string">
      Capisum transaction ID, prefixed with `txn_`.
    </ResponseField>

    <ResponseField name="status" type="string">
      Current lifecycle status: `pending`, `processing`, `successful`, `settled`, or `failed`.
    </ResponseField>

    <ResponseField name="type" type="string">
      Operation type: `pay_in` or `payout`.
    </ResponseField>

    <ResponseField name="amount" type="integer">
      Transaction amount in the smallest unit of the currency.
    </ResponseField>

    <ResponseField name="currency" type="string">
      ISO 4217 currency code.
    </ResponseField>

    <ResponseField name="provider" type="string">
      Provider slug that executed the transaction.
    </ResponseField>

    <ResponseField name="reference" type="string">
      Your unique reference string for this transaction.
    </ResponseField>

    <ResponseField name="created_at" type="string">
      ISO 8601 datetime string indicating when the transaction was created.
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="total" type="integer">
  Total number of transactions matching the applied filters, across all pages. Use this to calculate page counts or display result totals in your UI.
</ResponseField>

<ResponseField name="limit" type="integer">
  The page size applied to this response, reflecting the `limit` you passed (or the default of `20`).
</ResponseField>

<ResponseField name="next_cursor" type="string | null">
  An opaque cursor string pointing to the next page of results. Pass this value as the `cursor` query parameter in your next request. Returns `null` when you have reached the last page.
</ResponseField>

<ResponseField name="has_more" type="boolean">
  `true` if additional pages of results exist beyond the current page; `false` when this is the final page. Always check this field before making a follow-up request.
</ResponseField>
