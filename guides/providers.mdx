Use [`GET /v1/providers`](/api-reference) to list provider slugs Capisum can route to. Connect credentials in the [dashboard](https://app.capisum.com) before creating transactions. See the [provider network](/guides/provider-network) for the broader integration landscape.

<Note>
  The API returns the provider **catalog**, not your connected providers. Use [`GET /v1/providers/{slug}/status`](/api-reference) or the dashboard to confirm a provider is connected and healthy.
</Note>

## Available providers

| Provider | Slug | API today | NGN bank | GHS | KES |
| --- | --- | --- | --- | --- | --- |
| Fincra | `fincra` | Payouts | Yes | Bank + mobile money | Bank + mobile money |
| Paystack | `paystack` | Payouts | Yes | Bank + mobile money | Mobile money only |
| Flutterwave | `flutterwave` | Payouts | Yes | Bank + mobile money | Bank + mobile money |
| PalmPay | `palmpay` | Payouts | Yes | — | — |

All providers share the same Capisum payout state machine: queued submission, per-attempt provider transfer references, webhook matching, and verify-before-retry. Each adapter handles its own request format, status mapping, and webhook parsing.

### Fincra (`fincra`)

Cross-border payment infrastructure for collections, payouts, and virtual accounts across Africa.

| Operation | Status |
| --- | --- |
| Payouts | Available |
| Account verification | Available (NGN) |
| Balance lookup | Available |
| Pay-in | Not available |
| Refunds | Not available |

### Paystack (`paystack`)

Nigerian and African payment gateway with transfer APIs.

| Operation | Status |
| --- | --- |
| Payouts | Available |
| Account verification | Available (NGN) |
| Balance lookup | Available |
| Pay-in | Not available |
| Refunds | Not available |

Paystack reuses stored transfer recipients per organization credential to avoid duplicate beneficiary creation.

#### Disable Transfer OTP

Paystack payouts require Transfer OTP confirmation to be **disabled** on your Paystack account. This setting is Paystack-specific and applies globally, including the test environment.

1. Log in to the [Paystack Dashboard](https://dashboard.paystack.com).
2. Open **Settings** in the left menu.
3. Open the **Preferences** tab.
4. In **Transfers**, uncheck **Transfer OTP confirmation**.

### Flutterwave (`flutterwave`)

Pan-African payments platform with direct transfer APIs.

| Operation | Status |
| --- | --- |
| Payouts | Available |
| Account verification | Available (NGN) |
| Balance lookup | Available |
| Pay-in | Not available |
| Refunds | Not available |

#### Live setup

Before using Flutterwave through Capisum:

1. Connect your Flutterwave v4 credentials in the dashboard.
2. Whitelist **all** Capisum static outbound IPs in Flutterwave live settings.
3. Enable transfers on your Flutterwave account (required before payouts can work).

Full IP list and connect steps: [Flutterwave setup](/guides/flutterwave).

### PalmPay (`palmpay`)

Nigerian digital wallet and merchant payout APIs.

| Operation | Status |
| --- | --- |
| Payouts | Available (NGN bank only) |
| Account verification | Available (NGN) |
| Balance lookup | Available (NGN) |
| Pay-in | Not available |
| Refunds | Not available |

#### Live setup

Before using PalmPay through Capisum:

1. Generate an RSA key pair locally with OpenSSL and upload the public key to PalmPay.
2. Paste the matching private key into Capisum (encrypted at rest; never shown again).
3. Add PalmPay’s platform public key for webhook verification.
4. Whitelist **all** Capisum static outbound IPs in PalmPay.

Full key and IP steps: [PalmPay setup](/guides/palmpay).

Set `"provider": "<slug>"` on [`POST /v1/transactions/payout`](/api-reference) to route explicitly, or omit it to use your organization's routing rules. See [Payout integration](/guides/payouts) for destination fields and examples.

**Check provider health**

```bash
curl "$CAPISUM_BASE_URL/v1/providers/paystack/status" \
  -H "Authorization: Bearer $CAPISUM_API_KEY"
```

If no health data exists yet, the response is `{ "status": "unknown" }`.

See [Account verification](/guides/account-verification) for name enquiry and balance endpoints.

## Listing providers

```bash
curl "$CAPISUM_BASE_URL/v1/providers" \
  -H "Authorization: Bearer $CAPISUM_API_KEY"
```
