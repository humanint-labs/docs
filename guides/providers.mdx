Use [`GET /v1/providers`](/api-reference) to list provider slugs Capisum can route to. Connect credentials in the [dashboard](https://app.capisum.com) before creating transactions. See the [provider network](/guides/provider-network) for the broader integration landscape.

<Note>
  The customer API returns the provider **catalog**, not your connected providers. Use [`GET /v1/providers/{slug}/status`](/api-reference) or the dashboard to confirm a provider is connected and healthy.
</Note>

## Available providers

| Provider | Slug | Customer API today | NGN bank | GHS | KES |
| --- | --- | --- | --- | --- | --- |
| Fincra | `fincra` | Payouts | Yes | Bank + mobile money | Bank + mobile money |
| Paystack | `paystack` | Payouts | Yes | Bank + mobile money | Mobile money only |
| Flutterwave | `flutterwave` | Payouts | Yes | Bank + mobile money | Bank + mobile money |
| PalmPay | `palmpay` | Payouts | Yes | — | — |

All providers share the same Capisum payout state machine: queued submission, per-attempt provider transfer references, webhook matching, and verify-before-retry. Each adapter handles its own request format, status mapping, and webhook parsing.

### Fincra (`fincra`)

Cross-border payment infrastructure for collections, payouts, and virtual accounts across Africa.

| Operation | Status |
| --- | --- |
| Payouts | Available |
| Pay-in | Not available |
| Refunds | Not available |

### Paystack (`paystack`)

Nigerian and African payment gateway with transfer APIs.

| Operation | Status |
| --- | --- |
| Payouts | Available |
| Pay-in | Not available |
| Refunds | Not available |

Paystack reuses stored transfer recipients per organization credential to avoid duplicate beneficiary creation.

### Flutterwave (`flutterwave`)

Pan-African payments platform with direct transfer APIs.

| Operation | Status |
| --- | --- |
| Payouts | Available |
| Pay-in | Not available |
| Refunds | Not available |

### PalmPay (`palmpay`)

Nigerian digital wallet and merchant payout APIs.

| Operation | Status |
| --- | --- |
| Payouts | Available (NGN bank only) |
| Pay-in | Not available |
| Refunds | Not available |

Set `"provider": "<slug>"` on [`POST /v1/transactions/payout`](/api-reference) to route explicitly, or omit it to use your organization's routing rules. See [Payouts](/guides/payouts) for destination fields and examples.

**Check provider health**

```bash
curl "$CAPISUM_BASE_URL/v1/providers/paystack/status" \
  -H "Authorization: Bearer $CAPISUM_API_KEY"
```

If no health data exists yet, the response is `{ "status": "unknown" }`.

## Listing providers

```bash
curl "$CAPISUM_BASE_URL/v1/providers" \
  -H "Authorization: Bearer $CAPISUM_API_KEY"
```

<ResponseExample>

```json
{
  "data": [
    { "slug": "fincra", "name": "Fincra", "configurable": true },
    { "slug": "paystack", "name": "Paystack", "configurable": true },
    { "slug": "flutterwave", "name": "Flutterwave", "configurable": true },
    { "slug": "palmpay", "name": "PalmPay", "configurable": true }
  ]
}
```

</ResponseExample>

Use [`GET /v1/payouts/capabilities`](/api-reference) for the NGN-focused contract version and supported destination types.
