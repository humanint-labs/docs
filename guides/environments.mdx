Capisum separates **staging** and **production** across API keys, provider credentials, and transaction data.

## Staging

Use staging to build and test integrations without moving real money:

- API keys created with `environment: staging`
- Fincra sandbox credentials connected in the dashboard
- Transactions stored with `environment: staging`

## Production

Production keys route to live provider APIs:

- API keys created with `environment: production`
- Fincra live credentials connected in the dashboard
- Real funds move according to provider settlement rules

## Rules

| Rule | Detail |
| --- | --- |
| Keys are environment-bound | A staging key never accesses production transactions |
| Credentials are per environment | Connect Fincra separately for staging and production |
| Webhook URLs include environment | Use `/v1/webhooks/fincra/{slug}/staging` or `.../production` |

<Warning>
  Never use production API keys in client-side code, public repositories, or shared logs.
</Warning>

## Choosing an environment

1. Start in **staging** with sandbox provider credentials
2. Run end-to-end payout tests including verify and webhook delivery
3. Connect live provider credentials and create a **production** API key
4. Rotate staging keys if they were used in shared environments

Environment management (switching the dashboard view, team access per environment) is handled in the Capisum dashboard, not through the public API.

When you are ready to go live, follow the [Go to production](/guides/go-to-production) checklist.
