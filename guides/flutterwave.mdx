Connect Flutterwave in the [dashboard](https://app.capisum.com/providers) to use Capisum with your own Flutterwave v4 credentials.

<Note>
  Static outbound IPs and review copy live in `config/flutterwave-setup.json` and power the Capisum dashboard Flutterwave review step.
</Note>

## Whitelist Capisum static outbound IPs

Add every Capisum static outbound IP to your Flutterwave live IP allowlist. Capisum calls Flutterwave from these addresses:

```txt
152.55.176.240
152.55.177.181
162.220.232.250
162.220.232.251
162.220.232.252
```

Missing any IP returns `PROVIDER_IP_NOT_WHITELISTED`.

## Enable transfers

Enable transfers on your Flutterwave account before payouts can work. Contact Flutterwave support if API / direct transfers are not yet enabled for your business.

## Connect in Capisum

1. Open [Providers](https://app.capisum.com/providers) and connect Flutterwave.
2. Enter live Client ID, Client secret, Encryption key, and webhook secret hash.
3. Add the Capisum webhook URL in Flutterwave **Settings → Webhooks**:

```txt
https://capisum.com/webhooks/flutterwave
```

4. Complete the final **Review** step and whitelist the Capisum IPs shown there.

## Sandbox transfer scenarios

Flutterwave sandbox transfers need an `X-Scenario-Key` header (same pattern as card test scenarios). Capisum sets this automatically for sandbox credentials:

- Default: `scenario:successful`
- Override with payout metadata: `flutterwave_scenario` or `scenario`

```json
{
  "amount": 10000,
  "currency": "NGN",
  "provider": "flutterwave",
  "destination": {
    "type": "bank_account",
    "bank_id": "ng.access",
    "account_number": "0690000031"
  },
  "metadata": {
    "flutterwave_scenario": "failed"
  }
}
```

Allowed values: `successful`, `reversed`, `failed`, `account_verification_failed`, `transfer_amount_below_limit`, `transfer_amount_exceeds_limit`, `daily_transfer_limit_exceeded`, `monthly_transfer_limit_exceeded`, `disabled_transfer`, `duplicate_reference`, `insufficient_balance`, `payout_creation_failed`.

Live Flutterwave credentials never receive `X-Scenario-Key`.

See [Providers](/guides/providers), [Payouts](/guides/payouts), and [Errors](/guides/errors).
