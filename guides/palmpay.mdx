Connect PalmPay in the [dashboard](https://app.capisum.com/providers) to use Capisum with your own PalmPay Open API credentials.

<Note>
  Static outbound IPs and setup copy live in `config/palmpay-setup.json` and power the Capisum dashboard PalmPay review step.
</Note>

## Generate an RSA key pair locally

PalmPay verifies Capisum requests with **your** public key. Generate a 2048-bit RSA key pair in your terminal:

```bash
openssl genrsa -out private.pem 2048
openssl rsa -in private.pem -pubout -out public.pem
pbcopy < public.pem
```

1. Paste the copied public key into PalmPay API settings.
2. Paste the contents of `private.pem` into Capisum only.

Capisum validates the private key and credentials when you click **Continue**, encrypts the private key at rest, never returns it in the dashboard or public API, and uses it to sign PalmPay requests (MD5 + RSA-SHA1).

Keep a secure backup of the private key. After save, Capisum will not show it again.

## PalmPay platform public key

Separately, copy **PalmPay’s platform public key** from the PalmPay developer/API settings into Capisum (same connect screen, Webhooks section). Capisum uses that key to verify incoming webhook signatures — it is not the public key you uploaded to PalmPay.

## Whitelist Capisum static outbound IPs

Add every Capisum static outbound IP to your PalmPay IP allowlist. Capisum calls PalmPay from these addresses:

```txt
152.55.176.240
152.55.177.181
162.220.232.250
162.220.232.251
162.220.232.252
```

## Connect in Capisum

Open [Providers](https://app.capisum.com/providers) and connect PalmPay. Everything is on one screen:

1. Generate keys, enter Merchant ID / App ID, and paste your RSA private key.
2. Add the Capisum webhook URL and PalmPay’s platform public key:

```txt
https://capisum.com/webhooks/palmpay
```

3. Whitelist the Capisum outbound IPs shown on the same page, then click **Connect** (Capisum validates before saving).
