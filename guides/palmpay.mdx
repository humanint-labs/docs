Connect PalmPay in the [dashboard](https://app.capisum.com/providers) to use Capisum with your own PalmPay Open API credentials. The connect screen is intentionally short — use this guide for key generation, formats, and troubleshooting.

<Note>
  Static outbound IPs and shared setup copy live in `config/palmpay-setup.json` (kept in sync with the Capisum dashboard).
</Note>

## Setup overview

On the Capisum connect page you will:

1. Enter Merchant ID, App ID, your RSA private key, and PalmPay’s platform public key
2. For **live** only: whitelist Capisum static outbound IPs in PalmPay (not required for sandbox/test)
3. Click **Connect** (Capisum validates credentials before saving)

You do **not** configure a payout callback URL in PalmPay. Capisum automatically sends its webhook URL as `notifyUrl` on every [initiate merchant payment](https://docs.palmpay.com/#/en-us/pay-outs/initiate-merchant-payment) request. Merchants cannot override `notifyUrl`.

## Generate an RSA key pair locally

PalmPay verifies Capisum requests with **your** public key. Generate a 2048-bit RSA key pair in your terminal:

```bash
openssl genrsa -out private.pem 2048
openssl rsa -in private.pem -pubout -out public.pem
```

Copy the public key as **base64** for PalmPay (DER → base64, no PEM headers):

```bash
openssl rsa -pubin -in public.pem -outform DER | base64 | pbcopy
```

Copy the private key as **base64** for Capisum (PKCS#8 DER → base64, no PEM headers):

```bash
openssl pkcs8 -topk8 -inform PEM -outform DER -nocrypt -in private.pem | base64 | pbcopy
```

| Key | Where it goes | Format |
| --- | --- | --- |
| `public.pem` | Upload to PalmPay | **Base64 only** — no `BEGIN` / `END` lines |
| `private.pem` | Paste into Capisum | **Base64 only** — no `BEGIN` / `END` lines |

Capisum validates the private key when you connect, encrypts it at rest, never returns it in the dashboard or public API, and uses it to sign PalmPay requests (MD5 + RSA-SHA1).

Keep a secure backup of the private key. After save, Capisum will not show it again.

### Matching keys

If PalmPay returns a signature / sign error, the private key in Capisum usually does not match the public key uploaded to PalmPay. Confirm they are a pair (same modulus) before reconnecting.

## PalmPay platform public key

Separately, copy **PalmPay’s platform public key** from the PalmPay merchant portal → Open API / developer settings (shown near your App ID) into Capisum (Credentials section on the connect screen).

Capisum uses that key to verify incoming webhook signatures. It is **not** the `public.pem` you uploaded to PalmPay.

## Whitelist Capisum static outbound IPs (live only)

PalmPay documents IP allowlisting for **live** authentication. Sandbox/test credentials normally do not need Capisum IPs whitelisted.

Before going live, add every Capisum static outbound IP to your PalmPay **live** IP allowlist. Capisum calls PalmPay from these addresses:

```txt
152.55.176.240
152.55.177.181
162.220.232.250
162.220.232.251
162.220.232.252
```

If a **live** connection fails with an IP whitelist error, confirm all five IPs are saved in PalmPay live, then retry.

## Payout notifyUrl and remark

On every PalmPay payout Capisum sets:

| Field | Value |
| --- | --- |
| `notifyUrl` | Capisum webhook URL (`https://capisum.com/webhooks/palmpay`) — not merchant-overridable |
| `remark` | `metadata.remark` when provided, otherwise `Capisum payout` |

## Connect in Capisum

Open [Providers](https://app.capisum.com/providers) and connect PalmPay. The page sections match this order:

1. **Credentials** — Merchant ID, App ID, RSA private key instructions, private key, and PalmPay platform public key
2. **Outbound IPs (live)** — copy each Capisum IP into PalmPay live allowlist when going live

For live, confirm the IP whitelist checkbox and click **Connect**. Sandbox/test can connect without whitelisting IPs.

## API hosts

Capisum uses PalmPay Open API hosts on `palmpay-inc.com`:

| Environment | Host |
| --- | --- |
| Test / sandbox | `https://open-gw-sandbox.palmpay-inc.com` |
| Production / live | `https://open-gw-prod.palmpay-inc.com` |

Country is fixed to Nigeria (`NG`) for Capisum PalmPay payouts.

## API operations

| Operation | Status |
