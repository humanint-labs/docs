This guide uses the **customer API** at `/v1/*`. Dashboard setup (connecting a provider, creating keys) happens at [app.capisum.com](https://app.capisum.com). Learn about [provider coverage on capisum.com](https://capisum.com/providers) or read the [sandbox overview](https://capisum.com/sandbox) before your first payout.

## Prerequisites

- A payment provider connected for **staging** in the dashboard
- A **staging** API key (`cps_...`) from the dashboard

<Warning>
  Public API keys use the `cps_` prefix and Bearer auth. Do not use dashboard JWT tokens or `/api/v1/*` routes from your backend integration.
</Warning>

## 1. Store your API key

```bash
export CAPISUM_API_KEY="cps_your_key_here"
export CAPISUM_BASE_URL="https://api.capisum.com"
```

## 2. List available providers

```bash
curl -s "$CAPISUM_BASE_URL/v1/providers" \
  -H "Authorization: Bearer $CAPISUM_API_KEY"
```

<ResponseExample>

```json
{
  "data": [
    { "slug": "fincra", "name": "Fincra", "configurable": true }
  ]
}
```

</ResponseExample>

<Note>
  This returns the provider **catalog**, not your connected providers. Use `GET /v1/providers/fincra/status` or the dashboard to check connection health.
</Note>

## 3. Create a payout

```bash
curl -s -X POST "$CAPISUM_BASE_URL/v1/transactions/payout" \
  -H "Authorization: Bearer $CAPISUM_API_KEY" \
  -H "Content-Type: application/json" \
  -H "X-Idempotency-Key: payout-demo-$(uuidgen)" \
  -d '{
    "amount": "500000",
    "currency": "NGN",
    "provider": "fincra",
    "recipient": {
      "destination": "bank_account",
      "bank_code": "305",
      "account_number": "7068270185",
      "account_name": "Test Recipient"
    }
  }'
```

## 4. Check status

```bash
# By reference
curl -s "$CAPISUM_BASE_URL/v1/transactions/ref/txn_abc123" \
  -H "Authorization: Bearer $CAPISUM_API_KEY"

# Poll provider
curl -s -X POST "$CAPISUM_BASE_URL/v1/transactions/{id}/verify" \
  -H "Authorization: Bearer $CAPISUM_API_KEY"
```
